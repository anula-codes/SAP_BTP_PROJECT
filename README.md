# NovaBuild P2P Analytics - SAP CAP Capstone

**KIIT University · 2025–26 · SAP Certified Associate - Backend Developer (SAP CAP)**

A Procure-to-Pay analytics backend built with SAP Cloud Application Programming Model, deployed on SAP BTP Cloud Foundry.

---

## What It Does

Covers the full P2P cycle as an OData V4 API:

```
Purchase Requisition → Purchase Order → Goods Receipt → Invoice Verification
```

Includes a SAP Fiori Elements dashboard with live KPI tiles, PR approval workflow, and vendor performance analytics.

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | SAP CAP Node.js (`@sap/cds v7`) |
| Database | SQLite (local) · SAP HANA Cloud (production) |
| API | OData V4 — auto-generated by CAP |
| Auth | SAP XSUAA · OAuth2 via BTP |
| Frontend | SAP Fiori Elements |
| Deployment | SAP BTP Cloud Foundry · `mta.yaml` |
| CI/CD | GitHub Actions |
| Testing | Jest · 37 REST cases |

## Quick Start

```bash
npm install -g @sap/cds-dk
git clone https://github.com/anula-codes/SAP_BTP_PROJECT
cd novabuild-p2p-analytics
npm install
cds watch
```

Open **http://localhost:4004** — seed data loads automatically.

```bash
npm test    # run all tests
```

## Key Endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/p2p/PurchaseOrders` | GET / POST | PO management |
| `/p2p/PurchaseRequisitions({id})/approvePR` | POST | Approve a PR |
| `/p2p/PurchaseRequisitions({id})/rejectPR` | POST | Reject a PR |
| `/p2p/GoodsReceipts` | GET / POST | Post GRs (over-delivery blocked) |
| `/p2p/InvoiceVerifications` | GET / POST | Invoice management |
| `/p2p/OpenPOTrackerView` | GET | Live open PO list |
| `/p2p/getDashboardKPIs()` | GET | Aggregated KPIs |

Full endpoint list in `test/tests.http`.
